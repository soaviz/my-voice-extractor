API Compatibility Issue

프로덕션 전환 전 반드시 결정해야 하는 핵심 분기입니다.

Blocker
현재 불일치

프론트 일부 흐름은 기존 Node server.js 기준인 /api/analyze-video, /api/render-video 구조를 기대할 수 있습니다. Railway FastAPI는 upload, generate, status, download 기반 구조입니다. 현재 선택은 B안이므로 프론트 API adapter 작업 대신 Node backend 재배포를 진행합니다.

Option A: adapt frontend to Railway FastAPI

이번 단계에서는 진행하지 않습니다. FastAPI extractor API는 기존 Node endpoint와 직접 호환되지 않습니다.

Option B: redeploy Node server.js backend Selected

로컬에서 검증된 Node API를 Railway 또는 Render에 재배포해 프론트 변경 폭을 줄입니다.

Risk / Blockers

출시를 막을 수 있는 위험 요소입니다.

Watch

API mismatch

프론트가 Node API를 호출하는 상태에서 Railway FastAPI URL을 넣으면 분석/렌더링이 실패할 수 있습니다.

FFmpeg runtime

Node backend를 재배포하는 경우 Railway/Render 환경에 FFmpeg 실행 가능 여부를 다시 확인해야 합니다.

Production storage

로컬 uploads/outputs 저장소는 장기 운영용이 아닙니다. R2/Supabase Storage 전환 계획이 필요합니다.

Domain QA

로컬 모바일 테스트는 통과했지만 soaviz.com에서 실제 CORS, 파일 업로드, 다운로드 링크 검증이 필요합니다.
